## Этап 2. Формирование первичных гипотез

### Стартовая гипотеза 1: PSPNet с ResNet-50 backbone

**Описание гипотезы**  

На основе результатов EDA (сильный дисбаланс классов: фон ~90%, cat ~4-5%, dog ~4-6%)
была выбрана следующая стратегия:
1) Архитектура: PSPNet с ResNet-50 backbone

- PSPNet хорошо зарекомендовала себя для задач сегментации благодаря модулю Pyramid Pooling,
который позволяет учитывать контекст разных масштабов
- ResNet-50 обеспечивает хороший баланс между качеством и скоростью обучения
- Использование предобученных весов на ImageNet для ускорения сходимости

2) Функция потерь: Комбинация CrossEntropyLoss и DiceLoss

- CrossEntropyLoss (weight=1.0) - базовый loss для многоклассовой сегментации
- DiceLoss (weight=1.0) - добавлен для борьбы с дисбалансом классов, 
так как он фокусируется на пересечении предсказанных и истинных масок
- Такая комбинация позволяет лучше сегментировать мало представленные классы (cat и dog)

3) Аугментации:

- RandomFlip (prob=0.5) - горизонтальные отражения для увеличения вариативности
- PhotoMetricDistortion - изменение яркости/контраста для робастности к освещению
- Базовый набор аугментаций для стартового эксперимента, чтобы оценить "сырое" качество модели

4) Гиперпараметры:

- Batch size: 8 (оптимально для GPU с 12GB памяти)
- Learning rate: 0.001 (AdamW optimizer)
- Weight decay: 0.01 для регуляризации
- Epochs: 200 с валидацией каждые 5 эпох
- Input size: 256x256 (как в исходном датасете)
- Полиномиальное затухание LR: PolyLR с power=0.9

## Этап 3. Эксперименты по улучшению качества

### Эксперимент 1 

**Описание эксперимента**

Анализ базовой модели выявил две ключевые проблемы: недостаточная робастность из-за
минимальных аугментаций и дисбаланс классов (cat/dog сильно уступают фону). 
Для их решения в эксперименте 1 были усилены аугментации и увеличен вес DiceLoss.
В базовую модель добавлены вертикальные отражения (RandomFlip vertical), повороты 
(RandomRotate до ±30°), расширен диапазон фотометрических искажений (brightness_delta=32,
contrast_range=0.5-1.5) и внедрен RandomCutOut для регуляризации. Вес DiceLoss увеличен с
1.0 до 1.5 для основного head и до 0.6 для auxiliary head.

**Результаты обучения**

ссылка на конфиг: \practicum_work\src\work_dirs\pspnet_exp1\pspnet_r50_256x256_exp1_aug_weights.py

ссылка на clearml: https://app.clear.ml/projects/078603f73f5f477aaaae5a707dea12f7/experiments/6236f08a77f34843b3e6985748a462b2/output/execution

**Анализ качества**

На тестовой выборке модель достигла mDice 84.09%,
что на 1.92% выше базовой (82.17%). 
Наибольший прирост показал класс dog (+4.41%, с 70.20% до 74.61%),
класс cat улучшился незначительно (+0.84%, до 79.06%), фон вырос до 98.61%.
В лучших предсказаниях модель достигает почти идеальной сегментации: 
5 примеров с mDice >98.5%, где dog сегментируется с точностью 100%, cat >97%. 
В худших случаях (индексы 55, 23, 115, 46, 49) модель полностью пропускает животных 
(Dice=0%), хотя фон сегментирует отлично (>95%). 
Это указывает на проблему с мелкими или сильно замаскированными объектами.

ASPP с dilation rates (1,6,12,18) для мультимасштабного контекста, 
скип-соединения для восстановления деталей. Из эксперимента 1 сохранены 
усиленные аугментации и вес DiceLoss=1.5,
batch size уменьшен до 6, learning rate до 0.0005.

### Возможности для улучшения 

Анализ ошибок лучшей модели выявил несколько направлений для дальнейшего повышения качества:

Проблема 1: Пропуск мелких объектов. В худших случаях (индексы 11, 23, 46) модель полностью игнорирует животных, хотя фон сегментирует идеально. Это указывает на то, что мелкие объекты теряются на фоне. Решением может стать применение тестовых аугментаций (TTA) с мультимасштабным анализом или использование архитектур с механизмами внимания (SegFormer, Mask2Former), которые лучше выделяют мелкие объекты.

Проблема 2: Частичная сегментация. В индексах 77 и 73 модель распознает лишь фрагменты животных (cat=4.89%, dog=10.83%). Вероятно, это связано со сложными ракурсами или частичным перекрытием. Улучшить ситуацию помогут более агрессивные аугментации при обучении, имитирующие частичное затенение и необычные ракурсы.

Проблема 3: Дисбаланс между cat и dog. Несмотря на прогресс, класс dog (76.40%) все еще отстает от cat (79.98%). Целесообразно ввести взвешенную функцию потерь с увеличенным весом для класса dog или применить методы увеличения выборки (oversampling) для изображений с собаками.

Дополнительные рекомендации: Использование ансамбля моделей (PSPNet + DeepLabV3+) может сгладить ошибки каждой архитектуры. Также перспективно применение предобученных энкодеров на более крупных датасетах или дообучение на сложных примерах (hard example mining).
