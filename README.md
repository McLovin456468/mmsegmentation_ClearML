# Проект: Мультиклассовая семантическая сегментация

## 1. Назначение и цель проекта

Реализовать полноценный пайплайн обучения модели семантической сегментации: анализ датасета, подготовка бейзлайн-модели, проведение экспериментов по повышению качества.

**Целевая метрика:** `mDice` (mean Dice score)

**Критерий успеха:** mDice > 0.75 на тестовой выборке

---

## 3. Входные данные

### 3.1. Датасет

- Формат: мультиклассовая семантическая сегментация
- **Особенность:** датасет содержит ошибки в разметке
- Требуется самостоятельная оценка качества данных и выбор стратегии чистки

### 3.2. Инструменты

| Компонент | Назначение |
|-----------|------------|
| MMSegmentation | фреймворк для семантической сегментации |
| ClearML | логирование экспериментов |
| CVAT (опционально) | доразметка данных |

---

## 4. Этапы выполнения

### 4.1. Этап 1: Исследовательский анализ данных (EDA)

**Задачи:**

| № | Задача | Результат |
|---|--------|-----------|
| 1 | Оценить качество датасета, выявить ошибки разметки | список проблемных семплов |
| 2 | Выбрать стратегию чистки (доразметка / удаление) | обоснованное решение |
| 3 | Провести EDA: баланс классов, распределение размеров изображений, распределение размеров объектов | графики, статистики |
| 4 | Оформить результаты в отчёте | раздел в README.md |

**Выходные артефакты:**
- Описание процедуры чистки с примерами (до/после)
- Графики и статистики EDA
- Вспомогательный код в `practicum_work/src/data/`

---

### 4.2. Этап 2: Формирование первичных гипотез (бейзлайн)

**Задачи:**

| № | Задача | Результат |
|---|--------|-----------|
| 1 | Сформулировать 1–2 стартовые гипотезы | описание в отчёте |
| 2 | Выбрать модель, гиперпараметры, loss-функцию, аугментации | обоснование выбора |
| 3 | Запустить обучение | ссылки на ClearML и конфиги |
| 4 | Проанализировать качество (метрики, примеры успехов/ошибок) | анализ в отчёте |

**Структура описания гипотезы:**

```markdown
### Стартовая гипотеза N

**Описание гипотезы**
- логика выбора модели
- логика выбора loss-функции
- логика выбора гиперпараметров
- логика выбора аугментаций

**Результаты обучения**
- ссылка на конфиг
- ссылка на ClearML

**Анализ качества**
- метрики
- примеры фейлов
- примеры корректных предсказаний
```

**Требования:**
- Все ссылки на ClearML — **публичные**
- Стандартные имена графиков сохранены

---

### 4.3. Этап 3: Эксперименты по улучшению качества

**Ограничения:**
- 2–4 эксперимента
- В один эксперимент — одно инкрементальное изменение

**Направления для экспериментов:**

| Тип изменения | Пример |
|---------------|--------|
| Аугментации | добавление сложных аугментаций |
| Архитектура | замена backbone, добавление модулей |
| Гиперпараметры | изменение LR, scheduler, batch size |
| Loss-функция | замена на комбинированный loss |
| Постобработка | CRF, морфологические операции |

**Структура описания эксперимента:**

```markdown
### Эксперимент N

**Описание эксперимента**
- какая проблема решается
- как предлагается её решить
- визуализация изменений (если применимо)

**Результаты обучения**
- ссылка на конфиг
- ссылка на ClearML

**Анализ качества**
- метрики
- примеры фейлов
- примеры корректных предсказаний
```

### 4.5. Этап 5: Оформление проекта и документация кода

**4.5.1. Структура репозитория**

Проект выполняется в форке от MMSegmentation. Корневой `README.md` заменяется отчётом по шаблону.

Структура сабмодуля `practicum_work`:

```
practicum_work
├── src
│   ├── data/                 # вспомогательный код для EDA и конвертации
│   │   ├── mmsegmentation_to_coco.py   # конвертация в COCO для CVAT
│   │   ├── coco_to_mmsegmentation.py   # обратная конвертация из COCO
│   │   └── eda.py                      # генерация графиков для EDA
│   └── analysis/             # скрипты для анализа качества
│       ├── dump_model_predictions.py   # сохранение предсказаний модели
│       └── save_best_worst_predictions.py  # примеры лучших/худших предсказаний
├── supplementary
│   └── viz/                  # визуализации для отчёта (графики, скриншоты)
```

**4.5.2. Документация кода**

В `README.md` под заголовком **«Этап 5. Документация кода»** размещается:

- схема структуры `practicum_work`
- краткое описание назначения каждого скрипта

**Пример оформления:**

```markdown
## Этап 5. Документация кода

practicum_work
├── src
│   ├── data
│   │   ├── mmsegmentation_to_coco.py 
│   │   │   - конвертация в формат COCO для загрузки в CVAT
│   │   ├── coco_to_mmsegmentation.py 
│   │   │   - конвертация в формат MMSegmentation из COCO
│   │   ├── eda.py 
│   │   │   - генерация графиков для EDA (распределение классов, размеров)
│   ├── analysis 
│   │   ├── dump_model_predictions.py 
│   │   │   - сохранение предсказаний модели на датасете
│   │   ├── save_best_worst_based_on_individual_dice_score.py
│   │   │   - сохранение примеров лучших и худших предсказаний
```

## 6. Примечания

- Датасет содержит ошибки разметки — требуется самостоятельная чистка или доразметка.
- Все эксперименты логируются в ClearML со стандартными именами графиков.
- Код должен быть документирован и организован в соответствии с указанной структурой.
